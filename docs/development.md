## External Cargo patch

Clone the three source candidates anywhere outside the Partizan clone. Create a
temporary Cargo home and write this uncommitted configuration, replacing the
paths with absolute paths:

```toml
[patch."crates-io"]
bitmesh = { path = "/absolute/path/to/bitmesh" }
thermograph = { path = "/absolute/path/to/thermograph" }
astralbase = { path = "/absolute/path/to/astralbase" }
```

The quoted registry key is significant. With `CARGO_HOME` pointing to that
temporary directory:

```bash
cargo generate-lockfile --manifest-path engine/Cargo.toml --offline
cargo test --manifest-path engine/Cargo.toml --locked --offline --all-targets
python3 -m venv /tmp/partizan-venv
/tmp/partizan-venv/bin/python -m pip install maturin
CARGO_NET_OFFLINE=true /tmp/partizan-venv/bin/python -m pip install \
  --no-build-isolation --no-deps .
```

The Maturin install inherits Cargo's external patch. No `.cargo/config.toml` is
committed, and no sibling-directory layout is required.

For direct Cargo commands, the same patch may be supplied without a file:

```bash
cargo test --manifest-path engine/Cargo.toml --offline \
  --config 'patch."crates-io".bitmesh.path="/absolute/path/to/bitmesh"' \
  --config 'patch."crates-io".thermograph.path="/absolute/path/to/thermograph"' \
  --config 'patch."crates-io".astralbase.path="/absolute/path/to/astralbase"'
```

## Dataset regeneration

Checked artifacts never depend on `/private/tmp`. New generated artifacts go
under the ignored `artifacts/generated/` directory unless `--output` is given.
Astralbase is always explicit:

```bash
python engine/orchestrator.py sample-label-shard \
  --astralbase-dir /absolute/path/to/astralbase
```

The runner invokes the generator twice, requires byte identity, validates the
JSONL, hashes it, and writes a manifest. Promoting those generated bytes into
`docs/` or `data/` requires an intentional review and claim-ledger update.
